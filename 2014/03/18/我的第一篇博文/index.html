博客建立的前因后果

去年我在BAE上做了一个小网站，虽然除了给一个同学写了个祝生日快乐的小网页之外，几乎没别的作用，但是还是感觉有一个网站可以折腾是挺有意思的一件事。

BAE开始收费后，我就果断放弃了它，然后寻找到了免费而强大的 GitHub，之后又发现了免费而强大的 hexo，再加上免费而强大的 MarkDown ，我的网站就这样诞生了。

建立博客用到的几个东西

1. Github

在这个网站GitHub.io，可以找到简单的教程，根据网页上的步骤，可以迅速的建立起一个属于我的网站。当然，是一个没有数据库、只有静态网页的网站。这已经足够建立一个博客了。

在进行上述过程中，需要学会Git工具的使用。因为在大学里学过Linux的课程，所以终端式的 Git Bash 我很快就入门了。

其实只需要记住四句话

当然我后来发现 GitHub for Windows 非常方便，如果不喜欢敲一堆命令的话，用这个也一样。

2. Hexo

之后我需要一个生成静态网页的工具。Github使用的是 jekyll ，而有网友推荐jekyll的加强版 octopress ，但是因为这两个都是需要Ruby环境的，所有我选择了以 Node.js 开发的hexo。使用hexo的步骤：

  1. 下载并安装Node.js。官网下载，傻瓜式安装。
  2. 使用 npm install -g hexo ，安装hexo
  3. 几个记不住的命令和一堆记不住的配置文件。详情请见某个大神的博客里的一篇文章hexo系列教程：（二）搭建hexo博客。

其实我也是刚刚才开始用hexo，几乎完全不会。不过即使这样，我的博客也像模像样了。

3. MarkDown

Markdown 是一种方便记忆、书写的纯文本标记语言，用户可以使用这些标记符号以最小的输入代价生成极富表现力的文档：譬如您正在阅读的这份文档。它使用简单的符号标记不同的标题，分割不同的段落，粗体 或者 斜体 某些文字，更棒的是，它还可以

  • 高亮一段代码：
  • 甚至是生成一个表格：
英雄 定位 真正定位
撼地神牛 强力团控 死成狗的ATM
幻影长矛手 推塔能力最强的后期 拖队友后腿
军团指挥官 稳定强力的先手控 野区吸血鬼

总之是一个方便又好用的东西。你可以在这个网站上在线编辑测试，也可以在这个网站里找到MarkDown的语法资料。

其他一些东西

我可能会写写日记，写写学习到的各种技术，当然也有可能玩腻了就不管了。在我查资料的过程中，有个博主全面的介绍了使用github搭建个人博客的方法，在下面的留言中，有很多人反对博主这么做。他们认为github的用处本应是代码托管和资源分享，不应该用来建个人博客。而一旦有更多的人知道可以在github上建博客，有可能造成很多像我一样的人去建立博客，却不进行资源的分享，进而使github不得不考虑封掉大陆的ip。具体的内容我忘了，反正我觉得我这么一个小博客是占不了github多少资源的，与其担心github封掉大陆的ip，还不如担心大陆封掉github的ip。

最后，一句话与所有程序员共勉：

优秀的代码是它自己最好的文档。当你考虑要添加一个注释时，问问自己，“如何能改进这段代码，以让它不需要注释？”
-Steve McConnell
